### 1.1 What is Spring?
Spring is a container (the Spring application context) that creates and manages application components called beans. It wires these beans together using Dependency Injection (DI). Instead of objects creating their own dependencies, the container creates them and injects them where needed.

Historically, XML was used for configuration:

```xml
<bean id="inventoryService"
 class="com.example.InventoryService" />
<bean id="productService"
 class="com.example.ProductService" />
 <constructor-arg ref="inventoryService" />
</bean>
```

Modern Spring uses Java-based configuration, which is type-safe and easier to refactor:

```java
@Configuration
public class ServiceConfiguration {
 @Bean
 public InventoryService inventoryService() {
 return new InventoryService();
 }
 @Bean
 public ProductService productService() {
 return new ProductService(inventoryService());
 }
}
```

**Spring Boot** introduces autoconfiguration. It automatically configures components based on what it finds on the classpath, significantly reducing the code required to set up an application.

### 1.2 Initializing a Spring application
To build the "Taco Cloud" application, the text recommends using the Spring Initializr to bootstrap the project structure. This can be done via the web (start.spring.io), command line, or IDEs.

### 1.2.1 Initializing a Spring project with Spring Tool Suite
In Spring Tool Suite, you create a project by selecting **File > New > Spring Starter Project**.
1.  Fill in project details (Maven coordinates).
2.  Select dependencies (e.g., Web, Thymeleaf, DevTools).
3.  The wizard connects to the Spring Initializr to generate and download the project.

### 1.2.2 Examining the Spring project structure
The project follows a standard Maven structure:
*   `src/main/java`: Application source code.
*   `src/test/java`: Test code.
*   `src/main/resources`: Non-Java resources.
*   `mvnw`: Maven wrapper scripts. 
	* This scripts allow you to **build your project** even if you do not have Maven installed on your machine.
	* They act as a wrapper to execute Maven commands directly from the project directory.
*   `application.properties`: Configuration file.

**BOOTSTRAPPING THE APPLICATION**
The main class uses `@SpringBootApplication`, which combines three annotations: `@SpringBootConfiguration` (configuration), `@EnableAutoConfiguration` (autoconfiguration), and `@ComponentScan` (component discovery). The `main()` method launches the application.

```java
package tacos;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
@SpringBootApplication
public class TacoCloudApplication {
 public static void main(String[] args) {
 SpringApplication.run(TacoCloudApplication.class, args);
 }
}
```
